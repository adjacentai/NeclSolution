Pickachu Bot: How We Built a Profitable Telegram Image Editor with Gemini 3 Pro

There is a difference between occupying territory and holding it. Any developer can build a Telegram bot and plant a flag. But building one that generates revenue — that sustains itself, grows, and defends its position — requires the discipline of a commander who understands supply lines, not just swords.

Pickachu Bot is our campaign study in Telegram product monetization. A consumer-facing AI image editor that handles everything from background removal to creative compositions, powered by Google's most capable image model. It does not merely exist. It profits.

This article is not a product showcase. It is a transparent field report — the unit economics, the technical decisions, and the strategic lessons learned.

The Product: What Pickachu Bot Does

Pickachu Bot is an AI image editor that lives entirely inside Telegram. Users send photos, describe what they want via text or voice, and receive professionally edited images.

The Interaction Flow

  1. Send a photo — or up to 14 photos as references
  2. Describe the edit — "Remove background, add gradient" or record a voice message
  3. The bot processes — Gemini 3 Pro generates the result
  4. Receive the output — preview (compressed) + original (lossless PNG/JPG)
  5. Take action — retry with different prompt, download, or start new

The entire interface is a single Telegram message that updates in place. No chat spam. Clean, decisive UX.

Voice Commands: The Killer Feature

Text prompts work for simple edits. But for complex descriptions, voice is superior:

"Take the product from the first photo, put it on the marble surface from the second photo, match the lighting from the third photo, and add a subtle shadow underneath."

Typing that takes 30 seconds. Speaking it takes 5. Voice commands doubled our engagement per user. The lesson: remove friction from the chain of command, and your troops move twice as fast.

Multi-Reference Support

Most image editing bots accept one image and one prompt. Pickachu accepts up to 14 reference images — the technical moat that competitors cannot easily storm:

  • "Combine elements from these 3 product photos into one hero image"
  • "Apply the style from this reference to my product photo"
  • "Make my photo match the aesthetic of these Instagram examples"

The Business Model: Credit-Based Monetization

Why Not Subscriptions?

Subscriptions create pressure to use the product even when you don't need it. For an image editing tool — where usage is naturally sporadic — subscriptions create a mismatch. Misalignment between a general and his army breeds desertion.

Why Credits?

Credits align cost with value. Heavy usage this month? Buy more. Quiet next month? Spend nothing. No waste, no guilt, no churn from unused commitments.

Technical Decisions

Single-Message UI

Standard Telegram bots send a new message for every interaction. We imposed order through a single-message architecture:

  • One message shows the current state (uploaded images, prompt, status)
  • Every action updates that same message instead of sending a new one
  • Users always see a clean, current command post

Cost Tracking per Request

Every generation logs its cost to the database:

request_id | user_id | model | prompt | status | cost_usd | created_at

This gives us:

  • Real-time unit economics monitoring
  • Per-user profitability analysis
  • Cost anomaly detection (if API pricing changes)
  • Data for pricing optimization

Localization

Full Russian and English support. Not just translated strings — the entire UX adapts: button labels, error messages, help text, onboarding flow. Adding languages is modular — new languages require only a new locale file, no code changes.

Lessons Learned

1. Voice Input > Text Input for Complex Edits

We expected 80% text, 20% voice. Reality: 45% text, 55% voice. The strategist who listens to how his army actually moves — rather than how he expected them to move — adapts and prevails.

2. Credit System > Subscription for Sporadic Use

Initial prototype used subscriptions. Churn after first month: 70%. Switching to credits: 85% repurchase within 60 days.

3. Quality Is Table Stakes — UX Is the Differentiator

Multiple bots deploy the same AI models. The difference is the experience of wielding them. Our single-message interface, voice commands, and multi-reference support create a noticeably superior operation.

Frequently Asked Questions

How long does a generation take?
5-15 seconds depending on complexity and server load.

What's the maximum image resolution?
Input: any resolution (auto-compressed for API). Output: up to the model's maximum (typically 1024×1024 or higher for Gemini 3 Pro).

Can users request specific aspect ratios?
Yes. "Make it square," "make it 16:9," "make it vertical for Instagram Stories" — all supported via natural language.

How do you handle abuse?
Content moderation via Gemini's built-in safety filters + custom rules. Usage rate limiting to prevent API abuse. User blocking for violations.
